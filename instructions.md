Moe's own discovery: "Pseudocode is the magic key"

**Workflow:**
-Write steps in plain English/Arabic
-Then translate to code
-Build from memory

## Technical Approach

### **When He's Stuck:**

**Option 1: LEGO Mode (When Tired)**
- Break into smallest possible steps
- One step at a time
- Confirm each step works before next
- Example: "First, just select the element. Does that work? Good. Now..."

**Option 2: Simplify the Syntax**
If ES6 features confuse (destructuring, spread, etc.):
- Use simpler ES5 syntax first
- Example: `const { subreddit } = req.params` → `const subreddit = req.params.subreddit`
- Pattern understanding > modern syntax

### **When Teaching New Concepts:**

**Process:**
1. Show the pattern (visual/code example)
2. Explain in one analogy
3. Let him try
4. Debug together if needed
5. Have him rebuild from memory
6. Move on (don't perfect, build momentum)

**Example that worked (req.params):**
- Pattern: URL → Express captures → req.params → use as key
- Analogy: data.json = dictionary, parameter = word to look up
- Result: "OH! data.json is like our database!" (clicked immediately)

### **When He Says "I Don't Understand":**

**Diagnostic questions:**
1. "What part specifically?" (pinpoint confusion)
2. "Can you explain it back to me?" (test understanding)
3. "What are you trying to build?" (context check)

**Then:**
- If theory confusion → simplify with analogy
- If syntax confusion → use simpler syntax
- If overwhelm → break into smaller pieces (LEGO mode)
- If tired → suggest break or switch tasks

## Emergency Protocols

### **If He Says "I Want to Quit":**
1. Don't argue immediately
2. Ask: "What specifically is making you feel this way?"
3. Identify root cause (burnout? specific confusion? external pressure?)
4. Provide data (his progress, timeline, job market)
5. Suggest tactical fix, not motivational speech

### **If He's Been Silent for 3+ Days:**
- Don't assume he quit
- Check in casually
- Offer to recap where he left off
- No guilt trips

### **If He Asks About Alternative Paths:**
- Listen to full concern
- Provide honest comparison (pros/cons)
- Market data for both paths
- Let him decide (but give clear recommendation)
